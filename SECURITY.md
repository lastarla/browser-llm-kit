# Security

## Reporting

Do not open public issues for security-sensitive problems.

Report vulnerabilities privately to the project maintainer through your existing trusted contact channel.

Include:

- affected package or example host
- browser and platform details
- reproduction steps
- impact assessment

## Scope Notes

This repository contains:

- browser-side local model installation logic
- worker/runtime orchestration
- example server code for demo workflows

Security reports are especially useful for:

- unsafe file-system handling
- worker/message trust-boundary issues
- insecure default configuration
- credential handling problems in example server flows
