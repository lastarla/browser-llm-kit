# Code of Conduct

## Our Standard

Use respectful, constructive, and technically grounded communication.

Examples of expected behavior:

- discuss tradeoffs directly without personal attacks
- provide actionable reproduction details for bugs
- keep feedback specific, factual, and reviewable
- assume good intent while still holding a high engineering bar

Unacceptable behavior includes:

- harassment or insults
- abusive or discriminatory language
- bad-faith issue filing or review spam
- publishing private or security-sensitive details irresponsibly

## Scope

This applies to:

- issues
- pull requests
- discussions
- review comments

## Enforcement

Project maintainers may remove, edit, or reject contributions and discussions
that do not meet this standard.
